ChatNova — Multi-Provider AI Chat Platform

A unified interface for interacting with Google Gemini, Cerebras, and Groq LLMs. Features Multi-Modal RAG, web search, MCP agents, long-term memory, Generative UI, and audio transcription — all with BYOK security.

⚡

Multi-Provider Orchestration

Seamless switching between Gemini, Cerebras, and Groq via LangChain with unified streaming.

📄

Multi-Modal RAG

Upload PDF, DOCX, TXT, and images for session-bound context with hybrid vector+keyword search and reranking.

🌐

Web Search Integration

Real-time DuckDuckGo search with automatic query expansion, image search, and source synthesis.

🤖

MCP Agent Support

Model Context Protocol adapters for custom tool execution with real-time SSE streaming of agent actions.

📊

Generative UI

Dynamic charts (bar, line, pie), image galleries, and structured data rendered inline using Recharts.

📝

Long-Term Memory

AI-extracted user facts saved to PostgreSQL, selectively retrieved per conversation for personalized responses.

🎤

Audio Transcription

Voice input via Groq Whisper integration with browser MediaRecorder for in-chat speech-to-text.

🔒

BYOK Security

User-level API key encryption (Fernet), JWT cookies, token blacklisting, rate limiting, and input sanitization.

Key Flow States

Input

User enters text, selects model, toggles web search, attaches files. /commands for search/tools toggle.

Auth

JWT from HTTP cookie verified. Token checked against blacklist. Rate limit enforced (10/min production).

Context Assembly

Session history loaded. Custom instructions fetched. Memories retrieved via LLM relevance filter. RAG chunks via hybrid vector+keyword search + rerank.

Web Search

LLM generates 3 optimized queries. Parallel DuckDuckGo execution. Image search if needed. Results merged and deduplicated.

LLM Streaming

System prompt built with context. Standard or Agent mode. Tokens streamed via SSE as JSON events.

Post-Processing

Output moderated for harmful patterns. Facts extracted and saved to memory. Session title auto-updated. Source citations appended.

Render

Markdown rendered with react-markdown. Generative UI components parsed from JSON. Charts via Recharts. Image gallery for search results.

SSE Event Protocol

The streaming protocol uses Server-Sent Events with JSON-encoded data lines. The frontend streamChat function in utils/api.ts handles parsing.

Streaming Protocol
sequenceDiagram
    participant F as Frontend
    participant B as Backend

    F->>B: POST /chat/stream (SSE)
    Note over B: Create message record in DB
    B-->>F: data: {"type":"metadata","message_id":42}

    Note over B: Begin LLM streaming
    loop Token Chunks
        B-->>F: data: {"type":"content","content":"Hello"}
        B-->>F: data: {"type":"content","content":" world"}
    end

    Note over B: Extract memories
    B-->>F: data: {"type":"memory_saved","content":"User likes Python"}
    B-->>F: data: [DONE]

    Note over F: Render complete message
    ```

Agent Streaming Events

When using MCP agent mode (/chat/agent-stream), additional event types provide real-time tool visibility:

  • tool_start{ "tool": "search", "input": "...", "tool_call_id": "..." }
  • tool_end{ "output": "...", "tool_call_id": "..." }
  • contentText token chunks from LLM
  • errorError message if agent fails

Build & Run

Two deployment paths: Docker Compose for full stack or local development with services run individually.

Docker (Full Stack)

Build and start all services
docker-compose up --build
Services

Local Development

1. Start database
docker-compose up db -d
2. Run migrations
cd backend && uv run alembic upgrade head
3. Start backend
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
4. Start frontend
cd frontend && pnpm dev
